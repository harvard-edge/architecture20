# Contributing to Architecture 2.0

Welcome! We are incredibly excited you're interested in contributing. *Architecture 2.0* isn't just a book—it is a community movement toward fully agentic design loops for computer architecture. 

We envision this repository serving as a central, living hub for the tools, loops, and frameworks that make Architecture 2.0 possible.

## How to Contribute

### 1. Add your tool to the Living Catalog (Appendix D)
Have you built a surrogate model, a simulator API, an LLM workflow, or an RTL generator that fits into the Architecture 2.0 vision?
* **Action:** Go to our [Issues tab](https://github.com/harvard-edge/architecture20/issues) and click **New Issue**.
* Select the **🚀 Submit an Architecture 2.0 Tool** template.
* Fill out the form. 
* Our maintainers will review the tool and add it to `tools.yml`, which dynamically compiles into the book's Appendix D.

### 2. Share an Agentic Design Loop
If you have built an end-to-end design loop (e.g., combining an RL agent with gem5, or an LLM architect with an open-source synthesis flow), we want to feature it!
* **Action:** Open an issue with the label `loop-submission`. Include a link to your repository and a brief explanation of the 5-part loop state (State, Action, Rejection, Evidence, Commitment). We occasionally highlight these in community newsletters or future book revisions!

### 3. Join the Community
* **Discord:** Connect with other researchers and engineers building agentic hardware design tools on our Discord server (Link coming soon).
* **Hugging Face Hub:** We encourage the community to host their surrogate predictors and architecture world models on the [Hugging Face Hub](https://huggingface.co/harvard-edge). 

### 4. Book Corrections (Typos, Broken Links)
If you find a typo, a broken cross-reference, or formatting issue in the book text:
* Feel free to open a Pull Request directly against the `dev` branch.
* Book source files are located in the `synthesis/book/chapters/` directory and are written in Quarto Markdown (`.qmd`).

Thank you for helping us build the future of computer architecture!
